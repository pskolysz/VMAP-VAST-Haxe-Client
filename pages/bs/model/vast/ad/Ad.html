class bs.model.vast.ad.Ad

Available on all platforms

Within the element are one or more elements.Advertisers and video content publishers may associate an element with a line item video ad defined in contract documentation, usually an insertion order. These line item ads typically specify the creative to display, price, delivery schedule, targeting, and soon.

@author Piotr Skolysz

Instance Fields

var advertiser:String

the name of the advertiser as defined by the ad serving party. This element can be used to prevent displaying ads with advertiser competitors. Ad serving parties and publishers should identify how to interpret values provided within this element. As with any optional elements, the video player is not required to support it

var creatives:Array<Creative>

[Requierd] the container for one or more Creative elements

var description:String

a string value that provides a longer description of the ad

var errors:Array<Error>

a URI representing an error-tracking pixel; this element can occur multiple times

var extensions:Array<Xml>

XML node for custom extensions, as defined by the ad server. When used, a custom element should be nested under to help separate custom XML elements from VAST elements. The following example includes a custom xml element within the Extensions element.

var id:String

[Requierd] an ad server-defined identifier string for the ad

var impressions:Array<Impression>

[Requierd] the container for one or more Impression elements

var pricing:Pricing

Pricing provides a value that represents a price that can be used by real-time bidding (RTB) systems. VAST is not designed to handle RTB since other methods exist, but this element is offered for custom solutions if needed. If used, the following two attributes must be identified: - model: identifies the pricing model as one of “CPM”, “CPC”, “CPE”, or “CPV”. - currency: the 3-letter ISO-4217 currency symbol that identifies the currency of the value provided (i.e. USD, GBP, etc.…) If the value provided is to be obfuscated/encoded, publishers and advertisers must negotiate the appropriate mechanism to do so. When included as part of a VAST Wrapper in a chain of Wrappers, only the value offered in the first Wrapper need be considered.

var sequence:Int

a number greater than zero (0) that identifies the sequence in which an ad should play; all elements with sequence values are part of a pod and are intended to be played in sequence

var survey:String

a URI to a survey vendor that could be the survey, a tracking pixel, or anything to do with the survey. Multiple survey elements can be provided. A type attribute is available to specify the MIME type being served. For example, the attribute might be set to type=”text/javascript”. Surveys can be dynamically inserted into the VAST response as long as cross-domain issues are avoided.
